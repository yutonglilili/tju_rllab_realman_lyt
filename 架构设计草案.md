🧠 总体架构（一句话）

低频用 curobo 做全局/局部轨迹规划，高频按点执行；目标变化时打断并重规划

🏗️ 一、系统模块划分

你整个系统分成 4 个模块：

1️⃣ 感知 / 模型模块

职责：

输出目标物体的 pose（xyzrpy）

输入：

图像 / 指令

输出：

target_pose: np.ndarray  # shape (6,)


特点：

低频 or 中频（5~20 Hz）

可能抖动（必须后处理）

2️⃣ 规划模块（curobo）

职责：

从当前状态 → 目标 pose

输出一条 joint trajectory

包含：

IK

碰撞检测（自碰撞 + 障碍物）

输入：

current_joint
target_pose
scene_obstacles


输出：

trajectory: List[np.ndarray]  # joint list

3️⃣ 执行模块（Trajectory Executor）

👉 ⭐ 这是核心（你要自己写）

职责：

按顺序执行 trajectory

支持中断

控制发送频率

4️⃣ 机器人接口（RealmanEnv）

你已经有：

rm_movej_follow()
或
send_pose()

🔄 二、核心控制流程（主循环）
🧩 总流程（强烈建议照这个写）
初始化：
    env.reset()
    current_trajectory = None
    trajectory_idx = 0

循环（50Hz）：

    1. 获取当前状态
        current_joint = env.get_state()

    2. 获取最新目标
        target_pose = model_output

    3. 判断是否需要重新规划
        if 满足条件:
            trajectory = curobo.plan(...)
            current_trajectory = trajectory
            trajectory_idx = 0

    4. 执行当前轨迹
        if current_trajectory 存在:
            取下一个点执行

    5. sleep（20ms）

⚙️ 三、关键模块详细设计
🔑 1. 目标变化检测（非常关键）
🎯 目的：

避免频繁规划 or 不响应变化

✅ 实现：
def is_target_changed(new_target, last_target, threshold=0.02):
    pos_diff = np.linalg.norm(new_target[:3] - last_target[:3])
    rot_diff = np.linalg.norm(new_target[3:] - last_target[3:])
    return pos_diff > threshold or rot_diff > 0.1

🔑 2. 重规划触发条件（核心逻辑）
✅ 推荐条件：
需要重规划 if:

1. 当前没有轨迹
OR
2. 目标变化明显
OR
3. 轨迹执行完
OR
4. 执行过程中 IK 失败 / 碰撞风险
OR
5. 超过最大规划间隔（比如 0.5s）

🔑 3. curobo 规划调用策略
❗ 不要每帧调用
✅ 推荐：
plan_interval = 0.3 ~ 0.7 秒

示例：
if time.time() - last_plan_time > plan_interval:
    replan = True

🔑 4. Trajectory 执行器设计（重点）
数据结构：
current_trajectory: List[np.ndarray]
trajectory_idx: int

执行逻辑：
if trajectory_idx < len(current_trajectory):

    joint_target = current_trajectory[trajectory_idx]

    env.arm.rm_movej_follow(joint_target)

    trajectory_idx += 1

🔑 5. 中断机制（关键能力）
当目标变化：
current_trajectory = None
trajectory_idx = 0


👉 下一帧会触发重新规划

🔑 6. 执行频率控制
推荐：
control_rate = 50 Hz  # 20ms

注意：

不要太快（SDK扛不住）

不要太慢（会卡）

🔑 7. 轨迹下采样（非常重要）
curobo 输出可能很密：
100~500 个点

你应该：
trajectory = trajectory[::2]  # 每2个取1个


或：

max_points = 50


👉 否则：

执行时间过长

反应慢

🔑 8. 防止“卡住 >1s”
加 watchdog：
if time_since_last_plan > 1.0:
    强制重新规划

🧱 四、完整伪代码（可以直接改成代码）
env = RealmanEnv(async_mode=False)

last_target = None
current_trajectory = None
trajectory_idx = 0
last_plan_time = 0

while True:

    # 1. 当前状态
    obs = env.compute_observation()
    current_joint = obs["joint"]

    # 2. 模型输出
    target_pose = get_model_output()

    # 3. 判断是否需要重规划
    replan = False

    if current_trajectory is None:
        replan = True

    elif is_target_changed(target_pose, last_target):
        replan = True

    elif trajectory_idx >= len(current_trajectory):
        replan = True

    elif time.time() - last_plan_time > 0.5:
        replan = True

    # 4. 执行规划
    if replan:
        trajectory = curobo.plan(current_joint, target_pose)

        if trajectory is not None:
            trajectory = trajectory[::2]  # 下采样
            current_trajectory = trajectory
            trajectory_idx = 0
            last_plan_time = time.time()
            last_target = target_pose

    # 5. 执行轨迹
    if current_trajectory is not None and trajectory_idx < len(current_trajectory):

        joint_target = current_trajectory[trajectory_idx]

        env.arm.rm_movej_follow(joint_target)

        trajectory_idx += 1

    time.sleep(0.02)

🚧 五、后续可升级方向（你之后可以做）
🚀 1. 平滑过渡（避免抖动）

新轨迹前几个点和旧轨迹拼接

🚀 2. 速度限制

joint delta 限幅

🚀 3. 失败 fallback
if plan失败:
    保持当前姿态
    或 小幅 retreat

🚀 4. 局部 IK fallback（高级）

当规划太慢：

先 IK 动一下 → 再等完整规划

🎯 最终总结（给你一句最重要的话）

你现在这套系统，本质是：
“可打断的轨迹执行器 + 低频重规划”
